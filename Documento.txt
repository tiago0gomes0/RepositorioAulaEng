Feito Via GitHub Desktop - Tiago Prata e Bruno Costa

Um dos pontos ruins da versão desktop é a necessidade do pull a cada nova modificação no repositório, desta forma se duas pessoa fazem alterações ao mesmo tempo em um repositório, quando uma delas mandar o push a outra pessoa precisara fazer o pull antes de mandar seu push, desta forma fazendo um merge das modificações. Ja na versão do GitHub na web ao dar o commit, automaticamente ja acontece o push, por isso nao ocorre a necessidade de um pull no passo 9.
